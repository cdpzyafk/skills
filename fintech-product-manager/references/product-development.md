# 产品研发方法论

## PRD 标准结构

```markdown
# [产品名称] 产品需求文档

## 1. 产品概述
- 一句话定义：面向 [用户群]，提供 [收益类型]，通过 [底层机制] 实现
- 核心价值主张

## 2. 用户故事
- As [用户角色], I want [需求], So that [价值]

## 3. 产品规格
- 标的资产 / 期权方向 / 到期周期范围
- 行权价设定规则（ATM/OTM 比例）
- 收益计算公式
- 转换条件与交割规则

## 4. 数据依赖
- 实时报价来源（主/备）
- 结算价来源与时间点
- 非交易日/假期应急方案（必填）
- 异常处理（停牌/数据中断）

## 5. 风险披露要求
- 最大亏损场景（带数字示例）
- 不保本说明
- 流动性风险

## 6. 验收标准
- 功能验收：Happy path + 异常路径
- 数据验收：定价偏差阈值
- 合规验收：披露文本审查
```

## 期权类理财产品设计要点

```
看涨型（Sell Call）：
  APR = (期权权利金 / 本金) × (365 / 存续天数) × (1 - 平台手续费比例)
  转换价 = 行权价（通常为发行时价格的 103%–115%）
  转换触发：到期价 >= 行权价

看跌型（Sell Put）：
  APR 同上
  转换价 = 行权价（通常为发行时价格的 85%–97%）
  转换触发：到期价 <= 行权价
```

**行权价选择原则：**

| 产品目标 | 行权价建议 | 用户预期 |
|---------|-----------|---------|
| 高收益、高转换概率 | ATM（平值，0% OTM） | 愿意接受高概率转换 |
| 均衡型 | 5%–10% OTM | 标准双币主流方案 |
| 低收益、低风险 | 15%–20% OTM | 保守型用户 |

## 新标的资产接入检查表

在将新标的资产（如 TSLA 股票期权）接入双币产品前，必须逐项确认：

```
□ 期权风格：美式（需额外处理提前行权）or 欧式？
□ 结算方式：实物交割 or 现金结算？T+? 交割周期？
□ 交易时段：是否有休市时间？假期如何处理？
□ 公司事件：分红、拆合股、财报日历是否已纳入风控？
□ 数据成本：实时报价费用是否纳入产品成本核算？
□ 对冲渠道：通过哪家经纪商对冲？手续费率？
□ 到期日对齐：产品到期日是否与标准期权到期日对齐？
□ 非交易日备份：已配置 Pyth Feed 或 Massive 快照作为备源？
□ 多实例部署：若使用 Tiger Brokers，是否实现行情抢占逻辑？
```
