# 产品调研方法论

## Step 1：市场与监管摸底

在开始任何产品调研前，先回答这 5 个问题：

```
1. 目标用户群体是谁？所在地区受何监管？
2. 标的资产（股票/加密货币/商品）的交易时间、流动性如何？
3. 定价依赖哪些数据源？数据实时性和成本如何？
4. 对手方风险由谁承担？结算路径是什么？
5. 竞品是否已有类似产品？收益率/费率结构如何？
```

---

## Step 2：竞品拆解框架

| 维度 | 调研要点 |
|------|---------|
| **产品结构** | 期权类型（Call/Put）、行权价设定逻辑、到期周期 |
| **收益设计** | APR 范围、权利金来源、转换价计算方式 |
| **风险披露** | 最大损失场景、提前终止条款、流动性限制 |
| **技术实现** | 定价数据源、结算触发方式、对冲策略 |
| **用户体验** | 认购门槛、申购/赎回时间窗口、到期通知机制 |

---

## Step 3：可行性评估维度

```
技术可行性：
  □ 行情数据接口是否可用，延迟/成本是否可接受？
  □ 定价模型是否支持该期权类型（美式/欧式）？
  □ 结算价获取路径是否自动化，异常如何兜底？
  □ 非交易日备份数据源是否已配置？（缺此项视为技术方案不完整）

合规可行性：
  □ 目标用户所在地是否允许此类产品？
  □ 平台是否持有相关牌照或可通过合规架构规避？
  □ 产品说明书是否满足信息披露要求？

商业可行性：
  □ 对冲成本（期权权利金 + 经纪商手续费）vs 给用户收益 + 平台利润
  □ 最小可行规模（AUM）覆盖固定成本
  □ 用户需求强度：是否有差异化竞争优势？
```

---

## 输出格式

产品调研报告应包含：
1. **执行摘要**（100字，含核心结论）
2. **竞品对比表**（覆盖上述5个维度）
3. **可行性评估**（技术/合规/商业三维）
4. **建议与风险**（是否推进，主要风险点，下一步行动）
